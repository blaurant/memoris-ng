  <action>
Add layer-3 derived subscriptions to `network_detail/subs.cljs` (keep existing layer-2 subs unchanged):

1. `:network-detail/network` — derives from `:network-detail/data`, returns `(:network data)`. This gives the network map with `:network/name`, `:network/center-lat`, `:network/center-lng`, `:network/radius-km`.

2. `:network-detail/productions` — derives from `:network-detail/data`, returns `(:productions data)`. List of production maps.

3. `:network-detail/stats` — derives from `:network-detail/data`, returns a map:
   ```clojure
   {:total-capacity-kwc (:total-capacity-kwc data)
    :energy-mix         (:energy-mix data)
    :consumer-count     (:consumer-count data)
    :production-count   (count (:productions data))}
   ```
   Returns nil when data is nil.

4. `:network-detail/has-productions?` — derives from `:network-detail/productions`, returns `(seq productions)` truthy/falsy. Used to guard empty state rendering.

Use `:<-` signal function syntax for all derived subs (standard Re-frame pattern).
